# HR Knowledge Assistant
### Built with Snowflake Cortex Search (RAG) + Cortex Analyst (Text-to-SQL) + Streamlit

A full-stack GenAI application that lets you talk to both unstructured HR policy documents and structured employee data using plain English — no SQL knowledge required.

Built after completing **Building Generative AI Apps to Talk to Your Data** (Course 3, Snowflake GenAI Specialization, Coursera).

---

## What this project does

Two modes in one Streamlit app:

**Mode 1 — RAG on HR Policies (unstructured)**
Ask "How many sick leave days do I get?" → Cortex Search finds the relevant policy chunk → Cortex COMPLETE synthesises a clear answer. No hallucination — answers are grounded in your actual documents.

**Mode 2 — Text-to-SQL on Employee Data (structured)**
Ask "Who are the top earners in Engineering?" → Cortex Analyst generates SQL → queries the database → returns a table and chart. No SQL knowledge needed.

## Concepts from Course 3 demonstrated

| Concept | Where used |
|---|---|
| RAG (Retrieval Augmented Generation) | Cortex Search on HR policy docs |
| Embeddings | Auto-generated by Cortex Search service |
| Hybrid Search | Keyword + semantic combined in `SEARCH_PREVIEW` |
| Text-to-SQL | Cortex COMPLETE generates SQL from natural language |
| Unstructured data pipeline | Raw text → chunked → indexed → searchable |
| Streamlit frontend | Full UI deployed inside Snowflake |
| Application deployment | Runs natively in Snowflake, no external hosting |

---

## Why RAG beats fine-tuning for this use case

RAG retrieves real documents at query time — so when HR policy changes, you update one document and the AI immediately knows. Fine-tuning would require retraining every time a policy changes. For live enterprise data, RAG always wins.

## Why Text-to-SQL beats a dashboard

A dashboard shows you what the designer thought to show. Text-to-SQL lets anyone ask any question. A non-technical HR manager can ask "Which employees haven't taken any leave this year?" without knowing SQL.

---

## How to run

1. Run `01_setup.sql` → `02_seed_data.sql` in a Snowflake worksheet
2. Run `03_cortex_search_rag.sql` to create and test the search service
3. Run `04_cortex_analyst_text_to_sql.sql` to test Text-to-SQL queries
4. Deploy Streamlit: Projects → Streamlit → + New → paste `05_streamlit_app.py`
